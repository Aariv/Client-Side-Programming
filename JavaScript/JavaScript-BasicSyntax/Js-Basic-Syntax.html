JavaScript Basic Syntax

4.1 Comments

Commments are ignored by Javascript runtime but grately useful in explaining your codes to others. You should use comments liberally to explain or documnet your codes.

4.2 Whitespaces

Like C++/C/Java, Javascript ignores additional whitespaces (blanks, tabs, newlines). It is strongly recommend to use additional whitespaces

4.3 Expressions

An expression is a combination of variables, literals, operators and sub-expressions that can be evaluated.

4.4 Statement and Block

A statement is a single programming instruction. Semicolon is optional in Javascript. However, if semicolon is missing, you need to end the statement with a newline. Most developers strongly ending each statement with a semicolon.
